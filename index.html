# 創作者 分類 帳號類型 粉絲數 貼文數 互動率 發文頻率 自架站 優先
⟳ 載入資料中...
資料說明： 粉絲數與貼文數由 Apify instagram-profile-scraper 每週自動抓取，互動率為近 12 篇貼文平均（按讚+留言）/粉絲數估算。 發文頻率依據近 90 天貼文數計算：高頻 ≥ 12 篇/月、中頻 4–11 篇/月、低頻 < 4 篇/月。 優先等級：P1 = 無自架站 + 粉絲 > 3 萬 + 近期活躍，P2 = 粉絲 1–3 萬或有站，P3 = 其他。 帳號來源：以 Instagram hashtag 搜尋 #室內設計 #裝潢知識 #統包 #裝修 #老屋翻新 #居家佈置 等為主。
n8n 自動更新設定指南
展開 ▾
01
準備 Apify 帳號與 API Token
至 apify.com 註冊，進入 Settings → API → 複製 Personal API Token。 建議使用 Actor：apify/instagram-profile-scraper， 支援一次輸入最多 100 個帳號，自動回傳 followers、posts、bio、external_url 等欄位。 每次執行約消耗 $0.002–$0.005 USD/帳號。
02
建立 Google Sheet 作為帳號主表
欄位設計：username display_name category account_type active — 手動維護這份主表（新增/移除帳號、設定分類），n8n 每週從這裡讀取帳號清單，再去 Apify 拉最新數據。 這樣的設計讓你可以快速新增帳號而不需要修改 workflow。
03
n8n Workflow 架構（每週一自動執行）
Schedule Trigger（每週一 09:00）→ Google Sheet 讀取帳號清單 → Apify HTTP Request（批次抓取 profile 數據）→ Wait 60s → Apify GET 結果 → Code Node（計算互動率、發文頻率、優先等級）→ Google Sheet 寫入 data tab → Code Node（生成 JSON）→ GitHub / Google Drive 儲存 data.json → 完成（HTML 平台從 data.json 讀取）
04
互動率計算邏輯（Code Node）
Apify 的 instagram-profile-scraper 不直接回傳互動率，需使用 instagram-post-scraper 抓近 12 篇貼文的按讚+留言數，再除以粉絲數。 公式：ER = (avgLikes + avgComments) / followers × 100 建議 ER > 5% 標為「高」、2–5% 為「中」、< 2% 為「低」。 若不想抓貼文資料，可先以粉絲數估算（微網紅通常 ER 較高）。
05
HTML 平台更新方式
n8n 最後一步將 JSON 存入 GitHub Repo / Google Drive， HTML 平台透過 fetch('data.json') 讀取。 如果放在本地端，把 n8n 輸出的 JSON 直接嵌入 HTML 的 window.KOL_DATA 變數即可， 不需要額外的伺服器。 建議部署到 GitHub Pages 或 Cloudflare Pages（免費），讓整個團隊都能看到。
06
Workflow JSON 檔案
完整可匯入的 n8n workflow JSON 已在下方獨立輸出。 匯入方式：n8n → 右上角 ⋯ → Import from file → 選擇 n8n_kol_workflow.json。 匯入後替換 YOUR_APIFY_TOKEN、YOUR_GOOGLE_SHEET_ID 即可使用。
